  Переработала логика запуска игры:

      - загрузка вопросов убрана из Library
      - topicId и difficulty сохраняются в state при старте игры
      - загрузка вопросов теперь происходит только в Practice
      - функции startNewGame и restoreGameState вынесены в actions.ts

**Проблемы:**

- Изначально логика загрузки вопросов была распределена между Library и Practice, что усложняло поток данных
- При обработке ошибок терялась оригинальная ошибка API
- В Dashboard могли накапливаться event listeners при навигации

**Решения (или попытки):**

- Разделена ответственность: Library - выбор параметров, Practice - загрузка данных
- Использовано свойство Error.cause для сохранения оригинальной ошибки
- Упрощена логика обновления статуса сети (без подписки на события)

**Мысли / Планы:**

- При необходимости расширить getApiErrorMessage, чтобы различать типы ошибок (network / auth / server)
- Добавить к карточкам тем в библиотеке флаги/иконки пройдено/в процессе
