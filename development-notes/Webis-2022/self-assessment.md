## Описание проделанной работы

В рамках проекта я занимался разработкой SPA-приложения, начиная с настройки окружения и заканчивая реализацией игровой логики и интеграцией с backend.

На начальном этапе я настроил проект: подключил TypeScript, Vite, ESLint, Prettier, Sass, Husky, а также продумал структуру проекта с учетом масштабируемости. Далее была спроектирована архитектура приложения с разделением на UI, бизнес-логику и слой работы с данными.

Я реализовал интеграцию с Supabase: создал таблицы, настроил политики доступа (RLS) и разработал API layer — набор функций для работы с базой данных (получение данных, сохранение результатов, достижений и другой игровой информации). Также была написана edge function для клиентской функции getQuestions, которая на сервере проверяет наличие батча вопросов по выбранному топику: если данные уже есть в базе, они возвращаются, а если нет — происходит генерация вопросов с помощью LLM с последующим сохранением.

Основная часть работы была связана с разработкой логики игры: отображение вопросов и ответов, проверка правильности, начисление очков, переходы между раундами и реализация подсказок. Я реализовал интерфейс страницы practice. В последствии для оптимизации работы приложения полностью переработал логику обновления компонента — вместо полного перерендеринга теперь обновляется только изменяемый контент, что улучшило производительность и пользовательский опыт.

Далее я имплементировал страницу Results, которая отображает статистику игр пользователя.

Эта страница позволяет просматривать результаты по различным играм, фильтровать их по уровню сложности и анализировать свою статистику.

Также я разработал финальный экран игры с системой достижений: в зависимости от результата пользователь получает соответствующий бейдж, который сохраняется в базу данных.

В процессе разработки я занимался рефакторингом, исправлением багов и оптимизацией кода, а также настроил unit-тестирование с помощью Vitest.

Основные сложности были связаны с работой Supabase (в частности с настройкой RLS), проектированием архитектуры и переработкой логики без нарушения существующего функционала.

Помимо разработки, я выполнял роль тимлида: проводил code review, помогал команде решать технические проблемы, участвовал в исправлении конфликтов в Git и следил за качеством кода и общей архитектурой проекта.

В результате был реализован полноценный проект с продуманной архитектурой, игровой механикой, системой сохранения прогресса и достижений, а также стабильной интеграцией с backend.
